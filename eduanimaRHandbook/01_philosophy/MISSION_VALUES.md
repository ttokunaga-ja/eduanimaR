# Mission / Vision / Values

## 背景（なぜ今これを定義するか）
eduanimaR は、大学LMS等の学習環境で発生する「資料の探索」「理解の要点抽出」「学習の計画づくり」を支援するための、Chrome拡張機能およびWebアプリの提供を想定する。

本ハンドブックの他章（Strategy/Customer/Product/Goals）では戦略・具体仕様・優先度を扱い、本章では“どのような価値提供をし、何を守り、どんな判断基準で作るか”の原則のみを定義する。

## Mission（存在意義）
学習者が、配布資料や講義情報の中から「今見るべき場所」と「次に取るべき行動」を素早く特定できるようにし、理解と継続を支援する。

補足（意図）:
- 本サービスの主目的は「資料に基づく理解の支援」であり、評価・小テスト等の“自動正解提示”をゴールにしない。
- 本サービスは、学内規程・授業ルール等に反する利用（評価の不正な補助等）を前提とした設計・提供を行わない。

## Vision（到達したい状態 / North Star）
必要な情報が、必要なときに、必要な文脈で見つかり、学習者が自律的に学習を設計できる状態を当たり前にする。

North Star（指標の考え方）:
- 「探す時間が減り、理解に使う時間が増える」こと
- 「学習の次の一歩が明確になる」こと

## Product Principles（プロダクト原則）
本章では実装を規定しないが、判断の拠り所となる原則を定義する。

1. 学習支援を目的とする
	- 目的: 資料探索、要点の把握、学習計画、理解の補助。
	- 判断の問い: “ユーザーの理解が深まるか／次の行動が明確になるか？”
	- 非目的: 「提出物・試験等の成果物を自動生成して提出可能にする」ことを主目的にしない。

2. データ最小化と目的限定
	- 収集・保存・処理は必要最小限にし、目的外利用をしない。
	- 判断の問い: “このデータがなくても目的を達成できないか？”

3. 厳格なアクセス制御（ゼロトラスト志向）
	- 認証はSSO（OAuth）を前提とし、ユーザーごとにアクセス可能な資料・生成結果を厳格に分離する。
	- 初期リリースでは、Google / Meta / Microsoft / LINE のSSOを必須の入口として用いる。
	- Chrome拡張機能とWebアプリのどちらから操作しても、同一の認証・認可境界で分離が保たれることを前提とする。
	- 将来の科目内共有（同じ授業の友人/TA等）を想定する場合でも、「共有されるデータ」と「共有しない個人データ」を分離し、デフォルト非共有を守る。
	- 判断の問い: “誤って他ユーザーのデータが見える経路はないか？”

4. 追跡可能性と説明可能性
	- 重要な処理・アクセスは後から検証できる形で記録し、ユーザーが何が起きたか理解できるようにする。
	- 判断の問い: “問題発生時に原因を特定できるか？”

5. 透明性（ユーザーにとっての可視性）
	- 何を保存し、何に使い、どこへ送信されるかを明確にする。
	- 判断の問い: “ユーザーが驚く挙動になっていないか？”

6. 可搬性と将来互換
	- データは将来の移行（別DB/別ストレージ/別LLM）を阻害しない形で扱う。
	- 判断の問い: “特定ベンダー前提の前処理・形式に閉じ込めていないか？”

## Values（行動指針）
### V1. User Trust First（信頼が最優先）
ユーザーの信頼を損なう近道は選ばない。
- 判断の問い: “これをユーザーが知ったらどう感じるか？”

### V2. Secure by Default（安全がデフォルト）
後付けのセキュリティではなく、最初から漏えい・誤設定・誤利用を前提に設計する。
- 判断の問い: “最悪ケースで何が起こり、どう防ぐか？”

### V3. Clarity over Cleverness（賢さより明確さ）
ユーザー体験・運用・監査において、理解可能な構造を優先する。
- 判断の問い: “半年後の自分/他人が理解できるか？”

### V4. Evidence over Assumption（推測より根拠）
精度・速度・コストの議論は計測と検証に寄せる。
- 判断の問い: “その結論を支える観測データはあるか？”

### V5. Respect Content Rights（権利の尊重）
扱う資料の権利と取り扱い条件を尊重し、責任境界を明確にする。
- 判断の問い: “このデータを保持・加工してよい根拠が明確か？”

## 01章のスコープ（他章との責務分離）
- 本章で定義する: 目的、原則、価値観、信頼・安全に関する基本姿勢。
- 本章で定義しない: 実装詳細（技術選定・構成・DB設計・API・UI）、優先順位、顧客セグメント、ロードマップ、KPI。
